## 任务 1：修复 patterns.js 中的导入路径

**优先级**：高  
**文件**：`src/patterns.js:1`  
**工作量**：1 行更改

### 问题

导入使用了错误的相对路径：

```javascript
import { sanitizeCategory } from '../src/utils.js';  // ❌ 错误
```

这由于 Node.js 模块解析而工作，但在语义上是错误的且令人困惑。

### 解决方案

更改为正确的相对路径：

```javascript
import { sanitizeCategory } from './utils.js';  // ✅ 正确
```

### 验收标准

- [ ] 导入路径已更正
- [ ] 所有测试仍通过（`npm test`）
- [ ] 没有其他导入使用 `../src/` 模式

## 任务 2：在 config.js 中添加输入验证

**优先级**：中  
**文件**：`src/config.js`  
**工作量**：5-10 行

### 问题

配置加载在验证前直接访问嵌套属性，可能导致空值错误。

### 解决方案

为嵌套配置访问添加防御性检查：

```javascript
// 在加载配置时
const patterns = config?.patterns ?? {};
const exclude = patterns?.exclude ?? [];
```

### 验收标准

- [ ] 所有嵌套属性访问都有空值检查
- [ ] 空/未定义配置不会崩溃
- [ ] 现有测试仍通过

## 任务 3：改进错误处理

**优先级**：中  
**文件**：`src/engine.js`、`src/detector.js`  
**工作量**：10-15 行

### 问题

某些错误路径没有适当的日志记录或优雅降级。

### 解决方案

- 为检测失败添加结构化错误日志
- 确保错误不会破坏整个管道
- 在失败时回退到仅正则检测

### 验收标准

- [ ] 所有错误路径都有日志记录
- [ ] 检测失败不会破坏插件
- [ ] 测试覆盖错误场景

## 任务 4：优化模式编译

**优先级**：低  
**文件**：`src/patterns.js`  
**工作量**：5-10 行

### 问题

正则模式在每次调用时重新编译，而不是被缓存。

### 解决方案

在模块加载时一次性编译模式并缓存结果。

### 验收标准

- [ ] 模式只编译一次
- [ ] 性能可测量改进（或至少没有回归）
- [ ] 所有测试通过

---

## 任务 5：添加 JSDoc 注释

**优先级**：低  
**文件**：所有公共 API 文件  
**工作量**：20-30 行

### 问题

许多公共函数缺少 JSDoc 注释，使代码更难理解。

### 解决方案

为所有导出的函数添加 JSDoc 注释：
- `@param` 及其类型
- `@returns` 及其类型
- `@description` 简要说明

### 验收标准

- [ ] 所有导出函数都有 JSDoc
- [ ] 类型准确
- [ ] 描述清晰简洁
